# Команды бота

Все команды — текстовые (slash-commands Telegram). Отправляются пользователем в личный чат с ботом.

## Сводная таблица

| Команда         | Параметры         | Назначение                                            | В MVP |
|-----------------|-------------------|-------------------------------------------------------|:-----:|
| `/start`        | —                 | Приветствие, краткая инструкция, список команд.       |   ✅   |
| `/help`         | —                 | Подробная справка.                                    |   ✅   |
| `/models`       | —                 | Список доступных моделей + отметка активной.          |   ✅   |
| `/model`        | `<model_name>`    | Переключить активную модель для текущего пользователя.|   ✅   |
| `/prompt`       | `<text>` \| —     | Установить системный промпт; без аргумента — сброс.   |   ✅   |
| `/reset`        | —                 | Сбросить рантайм-настройки (модель, промпт) к дефолту.|   ⏳   |
| *любой текст*   | —                 | Отправить сообщение в LLM, получить ответ.            |   ✅   |

`⏳` — желательно, но не обязательно для MVP.

## Подробное поведение

### `/start`
- **Вход**: нет параметров.
- **Выход**: приветственное сообщение.
- **Шаблон ответа**:
  ```
  Привет! Я — AI-бот на локальной LLM (Ollama).

  Просто напиши мне что угодно — я отвечу.

  Команды:
  /help — справка
  /models — доступные модели
  /model <имя> — выбрать модель
  /prompt <текст> — задать системный промпт (без текста — сброс)
  ```

### `/help`
- **Вход**: нет параметров.
- **Выход**: расширенная справка, включая текущую модель и текущий системный промпт (обрезанный до N символов).

### `/models`
- **Вход**: нет параметров.
- **Выход**: список `OLLAMA_AVAILABLE_MODELS` с пометкой активной модели пользователя.
- **Пример**:
  ```
  Доступные модели:
  • qwen3.5:0.8b ← активная
  • deepseek-r1:1.5b

  Смени командой: /model deepseek-r1:1.5b
  ```

### `/model <model_name>`
- **Вход**: одно слово — имя модели.
- **Валидация**:
  - если аргумент отсутствует → подсказка «Использование: `/model <имя>`, список: /models».
  - если имя не в `OLLAMA_AVAILABLE_MODELS` → «Модель не найдена. Доступно: ...».
- **Эффект**: записать `user_id → model_name` в `model_registry` (in-memory).
- **Ответ**: «Модель переключена на `deepseek-r1:1.5b`.»

### `/prompt [<text>]`
- **Вход**: произвольный текст (может быть пустым).
- **Эффект**:
  - если текст непустой → установить системный промпт для пользователя (in-memory).
  - если пусто → сбросить к `SYSTEM_PROMPT` из `.env`.
- **Ответ**:
  - «Системный промпт обновлён.» / «Системный промпт сброшен к значению по умолчанию.»

### `/reset` (опционально)
- **Эффект**: удалить запись пользователя из `model_registry` и `prompt_registry`.
- **Ответ**: «Настройки сброшены.»

### Произвольный текст (не команда)
- **Вход**: любое текстовое сообщение, не начинающееся с `/`.
- **Алгоритм**:
  1. Отправить `ChatAction.TYPING`.
  2. Взять модель: `model = registry.get(user_id, settings.OLLAMA_DEFAULT_MODEL)`.
  3. Взять промпт: `system = prompt_registry.get(user_id, settings.SYSTEM_PROMPT)`.
  4. Вызвать `await llm_client.generate(text, model=model, system_prompt=system)`.
  5. При успехе — `message.answer(response)`.
     - если ответ длиннее 4096 символов — разбить на части (Telegram limit).
  6. При ошибке — ответить человекочитаемым сообщением (см. таблицу ошибок в `architecture.md`).

## BotFather / setMyCommands

При старте бота зарегистрировать команды для UI Telegram:

```python
await bot.set_my_commands([
    BotCommand(command="start", description="Начать работу"),
    BotCommand(command="help", description="Справка"),
    BotCommand(command="models", description="Список моделей"),
    BotCommand(command="model", description="Выбрать модель"),
    BotCommand(command="prompt", description="Задать системный промпт"),
])
```

## Ограничения ввода

- Максимальная длина входного текста (для запроса в LLM) — можно ограничить, например, 4000 символов. При превышении — сообщение «Слишком длинный запрос, сократите».
- Нетекстовые сообщения (фото, стикер, голос) в MVP — игнорируются или отвечают «Я понимаю только текст».
