## 1. Цель спринта

Подготовить локальное dev-окружение и базовый скелет репозитория, чтобы можно было приступить к написанию приложения. Никакого продуктового кода — только окружение, зависимости и каркас пакетов.

## 2. Скоуп и non-goals

- **В скоупе:** установка Python 3.11+, Ollama, выбор моделей; создание `.gitignore`, `.env.example`, `requirements.txt`, пустых пакетов `app/` и `tests/`.
- **Вне скоупа (non-goals):** конфигурация (`Settings`), логирование, LLM-клиент, handlers — всё это в спринте 01.

## 4. Этап 1. Подготовка окружения и скелета

Подготовка хост-окружения и каркаса репозитория. Без логики приложения.

### Задача 1.1. Подготовка окружения и зависимостей

- **Статус:** Done
- **Приоритет:** high
- **Объём:** S
- **Зависит от:** —
- **Связанные документы:** `_docs/instructions.md` §10, `_docs/stack.md` §1–§3, §10, `_docs/roadmap.md` «Этап 0».
- **Затрагиваемые файлы:** локальное окружение хоста (вне репозитория).

#### Описание

Подготовить локальное dev-окружение, необходимое для запуска проекта:

1. Установить Python 3.11+ (проверить `python -V`).
2. Установить Ollama (`https://ollama.com`), запустить сервис (`ollama serve`).
3. Загрузить модели из ТЗ (теги уточнить через `ollama list`):
   - `ollama pull qwen3.5:0.8b` (или аналогичный доступный тег семейства Qwen);
   - `ollama pull deepseek-r1:1.5b`.
4. Создать Telegram-бота через `@BotFather`, сохранить токен локально (пока **не** коммитить).
5. Создать виртуальное окружение в корне репозитория: `python -m venv .venv`, активировать.

#### Definition of Done

- [x] `python -V` выводит версию ≥ 3.11.
- [x] `ollama list` показывает обе модели (фактические теги зафиксированы в заметках — потребуются для `.env`).
- [x] Токен Telegram получен и сохранён локально (вне репозитория).
- [x] Каталог `.venv/` создан, активация работает, `pip --version` отрабатывает внутри окружения.
- [x] В репозитории **не появилось** артефактов от установки (`.env`, `.venv/` в `git status` не коммитятся).

### Задача 1.2. Скелет репозитория и базовые файлы

- **Статус:** Done
- **Приоритет:** high
- **Объём:** S
- **Зависит от:** Задача 1.1
- **Связанные документы:** `_docs/project-structure.md`, `_docs/stack.md` §8–§9, `_docs/roadmap.md` «Этап 1».
- **Затрагиваемые файлы:** `.gitignore`, `.env.example`, `requirements.txt`, `app/**/__init__.py`, `tests/**/__init__.py`, `README.md`.

#### Описание

Сформировать целевую структуру репозитория согласно `_docs/project-structure.md`:

1. Создать `.gitignore` со всеми блоками из `_docs/project-structure.md` § «Что должно попасть в `.gitignore`» (Python, Tests/tools, IDE, Logs, Secrets).
2. Создать `.env.example` с полным набором переменных из `_docs/stack.md` §9 (`TELEGRAM_BOT_TOKEN`, `OLLAMA_BASE_URL`, `OLLAMA_DEFAULT_MODEL`, `OLLAMA_AVAILABLE_MODELS`, `OLLAMA_TIMEOUT`, `SYSTEM_PROMPT`, `LOG_LEVEL`, `LOG_FILE`) с комментариями-подсказками.
3. Создать `requirements.txt` минимум с:
   - `aiogram>=3.4,<4`
   - `ollama>=0.3`
   - `pydantic-settings>=2.1,<3`
   - `pytest>=8`
   - `pytest-asyncio>=0.23`
   - `pytest-mock>=3.12`
4. Создать структуру пакетов с `__init__.py`:
   - `app/`, `app/handlers/`, `app/services/`, `app/middlewares/`, `app/utils/`
   - `tests/`, `tests/services/`, `tests/handlers/`
5. Создать заготовку `README.md` в корне (структура разделов: «Установка», «Настройка», «Запуск», «Команды бота», «Тесты»; наполнение — на этапе финального спринта).
6. Удалить заглушку `main.py` из корня (функциональность переедет в `app/main.py`).

#### Definition of Done

- [x] Структура каталогов полностью соответствует `_docs/project-structure.md` (включая тесты, зеркалирующие `app/`).
- [x] `pip install -r requirements.txt` в активированном `.venv` проходит без ошибок.
- [x] `git status` показывает только добавленные файлы, реальный `.env` отсутствует.
- [x] `.env.example` содержит **все** 8 переменных, указанных в `_docs/stack.md` §9.
- [x] `python -c "import app"` не падает (пакет импортируется).
