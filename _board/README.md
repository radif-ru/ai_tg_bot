# Доска задач (board)

Артефакты процесса для LLM-агентов: план задач, алгоритм выполнения, ad-hoc заметки. Документы здесь — **рабочее состояние**, в отличие от `_docs/`, где лежит **спецификация проекта**.

## Состав

- [`plan.md`](./plan.md) — детальный план задач со статусами `ToDo`/`Progress`/`Done` и Definition of Done.
- [`process.md`](./process.md) — пошаговый алгоритм выполнения **одной** задачи (универсален, не зависит от конкретной задачи).
- [`progress.txt`](./progress.txt) — ad-hoc заметки о прогрессе и фиксация чек-листа приёмки MVP.

## Связи с другими артефактами

- Спецификация проекта (требования, архитектура, стек) — в `../_docs/`. Подробнее: `../_docs/README.md`.
- Поведенческие гайдлайны LLM-агента — `../CLAUDE.md` (общие, не проектные).
- Точка входа для нового агента: `../CLAUDE.md` → `../_docs/README.md` → `plan.md` → `process.md`.

## Правила

1. В статусе `Progress` в `plan.md` одновременно — **ровно одна** задача.
2. Каждая смена статуса фиксируется атомарным коммитом `chore(plan): ...`.
3. Пошаговый порядок действий — в `process.md`. Отступать от него не следует.
